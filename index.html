Moodie — A mental wellness mobile application that lets users record their daily emotions, monitor mood patterns, and reflect on their mental well-being over time — built with Jetpack Compose and modern Android architecture.

Supporting SDG 3: Good Health and Well-Being, because Moodie promotes mental wellness by enabling users to track, understand, and reflect on their emotions regularly.

Labs & Learning Journey

Lab 1: Jetpack Compose UI Fundamentals

Exploring layouts with Column, Row, and Box components to build Moodie's home screen interface.

Reflection: This lab helped me understand the basics of Jetpack Compose layouts through the development of Moodie's home screen. I applied Column, Row, and Box components to arrange interface elements in a structured and user-friendly way. The experience improved my understanding of UI design and prepared me for future app development tasks.

Lab 2: Interactive Components & State Management

Adding TextField, Button interactions, and state management with remember and mutableStateOf.

Reflection: In Lab 2, I enhanced Moodie by adding interactive features using TextField, Button, and state management with remember and mutableStateOf. Users can enter information and receive immediate feedback on the screen, making the application more engaging and responsive. Through this lab, I gained a better understanding of how state works in Jetpack Compose and how dynamic user interactions improve the overall user experience.

Lab 3: Material Design & Visual Enhancement

Applying Material Design principles, custom themes, cards, and animations to Moodie.

Reflection: In Lab 3, I improved the Moodie application by applying Material Design principles, including a custom theme, cards, and animations. I selected colors that reflect mental wellness and created a more visually appealing interface using Card components to organize content. This lab helped me understand how themes, typography, and animations can enhance user experience and make an application look more professional.

Lab 4: Navigation & ViewModel Architecture

Implementing multi-screen navigation using Jetpack Compose Navigation and managing state with ViewModel.

Reflection: In Lab 4, I enhanced Moodie by implementing multiple screens using Jetpack Compose Navigation and managing user data with ViewModel. The app was upgraded to allow users to navigate smoothly between different pages while maintaining their mood-related information across screens. Through this lab, I gained a better understanding of application architecture, state management, and how ViewModel helps preserve data during configuration changes.

Lab 5: Room Database & Data Persistence

Integrating Room Database for persistent storage of mood records with Entity, DAO, and Repository patterns.

Reflection: In Lab 5, I integrated Room Database into Moodie to enable persistent storage of users' mood records. By implementing Entity, DAO, Repository, ViewModel, and Room Database components, I was able to save and retrieve mood data even after the application was closed and reopened. This lab helped me understand the MVVM architecture and the importance of data persistence in developing reliable mobile applications.

Projects

Project 1: Moodie Multi-Screen Application

Problem: Mental health awareness is becoming increasingly important, especially among students who often experience stress, anxiety, and emotional challenges in their daily lives. However, many individuals do not regularly monitor their emotional well-being, making it difficult to identify mood patterns and seek support when needed. Moodie provides a simple platform for users to record, track, and reflect on their moods.

Lessons Learned: Through Project 1, I learned how to build a complete multi-screen mobile application using Jetpack Compose, Navigation, and ViewModel. I gained experience in managing shared state across different screens while maintaining a consistent and user-friendly interface. This project strengthened my understanding of mobile application architecture and prepared me for more advanced features such as databases, APIs, and cloud integration in Project 2.

Resources & References
