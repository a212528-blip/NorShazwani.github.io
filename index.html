Hands-on practice

Lab Sessions

Five labs progressively building Moodie from a static screen to a data-driven, navigable application.

Lab 1

UI — Jetpack Compose Layouts

Designing Moodie's home screen using Column, Row, and Box to build a clean, structured interface from scratch.

This lab helped me understand the basics of Jetpack Compose layouts through the development of Moodie's home screen. I applied Column, Row, and Box components to arrange interface elements in a structured and user-friendly way. The experience improved my understanding of UI design and prepared me for future app development tasks.

Lab 2

Interaction — State Management

Adding TextField, Button, and Compose state with remember and mutableStateOf for real-time user feedback.

In Lab 2, I enhanced Moodie by adding interactive features using TextField, Button, and state management with remember and mutableStateOf. Users can enter information and receive immediate feedback on the screen, making the application more engaging and responsive. Through this lab, I gained a better understanding of how state works in Jetpack Compose and how dynamic user interactions improve the overall user experience.

Lab 3

Material Design — Themes & Animation

Applying a custom Material theme, Card components, and animations to create a polished, wellness-inspired visual identity.

In Lab 3, I improved the Moodie application by applying Material Design principles, including a custom theme, cards, and animations. I selected colors that reflect mental wellness and created a more visually appealing interface using Card components to organize content. This lab helped me understand how themes, typography, and animations can enhance user experience and make an application look more professional.

Lab 4

Navigation — Multi-Screen & ViewModel

Implementing Jetpack Compose Navigation and ViewModel so users move fluidly between screens while mood data persists across them.

In Lab 4, I enhanced Moodie by implementing multiple screens using Jetpack Compose Navigation and managing user data with ViewModel. The app was upgraded to allow users to navigate smoothly between different pages while maintaining their mood-related information across screens. Through this lab, I gained a better understanding of application architecture, state management, and how ViewModel helps preserve data during configuration changes.

Lab 5

Room — Persistent Local Storage

Integrating Room Database (Entity, DAO, Repository) so mood records survive app restarts using the full MVVM pattern.

In Lab 5, I integrated Room Database into Moodie to enable persistent storage of users' mood records. By implementing Entity, DAO, Repository, ViewModel, and Room Database components, I was able to save and retrieve mood data even after the application was closed and reopened. This lab helped me understand the MVVM architecture and the importance of data persistence in developing reliable mobile applications.
